# API

  • choice

    The Choice interface defines a Choice as it will be suggested to the user. It is used in the getChoices() function in any Rule implementation. Usually the getChoices() function returns an array of Choice.

    Properties

    name
    The string that will be shown in the terminal as a choice.
    value
    The value of the given choice. If it's a boolean, the apply method will be called or not, depending on if it's true or false. If it's a string, the string is passed to the apply.

    Constants

    Ok
    The Ok Choice. Always return true, it is used when you don't need to apply a rule for example.
    YesNo
    The Yes|No Choice. If the user chose Yes, it returns true so the apply method is called.

  • register

    The Register class that contains register decorators and other methods to retrieve registered Stacks and Rules

    Methods

    getAllStacks(): Stack[]
    Returns all registered Stacks as an Array of Stacks.
    getAvailableStacks(): Promise<undefined[]>
    Returns every Stacks that are available in the audited project.
    getRulesToApply(): Promise<undefined[]>
    Returns all the Rules that may be applied in the audited project.
    getStacksMaps(): StackMap[]
    Returns an array of objects containing an instanciated Stack in the stack property, and its associated sub Stacks in the subStacks property.
    getSubRulesOf(): Rule[]
    Returns an array containing all the sub Rules of a given Rule.
    ruleForAll(): (Anonymous function)<Anonymous function> Decorator
    Registers the decorated Rule for every Stacks, except for ones given in options Object.
    ruleForStacks(): (Anonymous function)<Anonymous function> Decorator
    Registers the decorated rule to the stacks given in parameter.
    stack(): undefined Decorator
    A class decorator that makes a class implement the Stack interface, and registers this class in an array.
    stackIsAvailable(): Promise<boolean>
    Returns a Promise of true if the given Stack is detected in the audited project.
    subRuleOf(): (Anonymous function)<Anonymous function> Decorator
    Registers a sub Rule of another Rule, so that the registered Rule will be called only after the parent Rule's apply has been succesful.
    subStackOf(): (Anonymous function)<Anonymous function> Decorator
    A classs decorator that registers a subStack of a given Stack

    • rule

      The interface that defines a Rule like NodeVersion, TSLint, GitIgnore..

      Methods

      apply(): Promise<void>
      Add what is necessary to the project so the rule should not be applied again. Takes a boolean or an array of string to tell if the rule has to be applied, or the choices to apply if the user has multiple choices to apply.
      getChoices(): undefined
      Returns a list of Choices for the user to chose. For Yes|No question, use native YesNo type, for just a confirmation, use Ok type.
      getLongDescription(): undefined
      Returns a longer description that will appear in the generated Markdown/PDF report.
      getName(): undefined
      Returns the name of the Rule as it will be seen in the prompt.
      getPromptType(): undefined
      Gives the prompt type as a string. Please refer to Inquirer documentation to know which prompt types exists. For Yes|No or confirmation, use "list" as prompt type, and use the "YesNo" or "Ok" types in getChoices() method.
      getShortDescription(): undefined
      Returns the description of the stack that will be seen before asking if the user wants to apply it.
      shouldBeApplied(): Promise<boolean>
      Resolves a promise to true if the audited project should apply this rule. For example, if the rule is to add rules to the .gitignore file, if the existing .gitignore does not contains all the rules, this method resolves to true.
